╔════════════════════════════════════════════════════════════════════════════════╗
║                            FLUJO DE DATOS PRINCIPAL                             ║
╠════════════════════════════════════════════════════════════════════════════════╣

1. VALIDACIÓN DE OC:
   └─→ main.py --oc OC123456
       ├─→ monitor.validar_oc_existente()
       ├─→ monitor.validar_distribuciones()
       ├─→ monitor.validar_asn()
       ├─→ monitor.validar_errores_excel()
       ├─→ si CRÍTICO → notificaciones_telegram.enviar_alerta()
       ├─→ gestor_correos.enviar_alerta_critica()
       ├─→ reportes_excel.crear_reporte_validacion_oc()
       └─→ guardar en output/resultados/

2. REPORTE DIARIO:
   └─→ main.py --reporte-diario
       ├─→ query_loader.load_query('oc_diarias')
       ├─→ db_connection.execute_query()
       ├─→ query_loader.load_query('asn_pendientes')
       ├─→ reportes_excel.crear_reporte_planning_diario()
       ├─→ gestor_correos.enviar_reporte_planning_diario()
       ├─→ notificaciones_telegram.enviar_resumen_diario()
       └─→ guardar en output/resultados/

3. MONITOREO EN TIEMPO REAL:
   └─→ monitor.py (ejecución continua)
       ├─→ cada 15 minutos
       │   ├─→ ejecutar queries preventivas
       │   ├─→ buscar errores CRÍTICO
       │   └─→ generar alertas
       ├─→ cada hora
       │   ├─→ sincronizar caché local
       │   └─→ actualizar dashboard
       └─→ dashboard.py actualiza en tiempo real

4. DASHBOARD WEB:
   └─→ python dashboard.py (http://localhost:5000)
       ├─→ db_local.DBLocal (caché)
       ├─→ datos actualizados cada 5 minutos
       ├─→ visualización de OCs
       ├─→ métricas en tiempo real
       └─→ histórico de validaciones
