1. ENTRADA DE USUARIO SIN VALIDACIÓN
   Archivo: main.py
   Líneas: 564, 635, 636, 686, 810, 813, 825, 826, 827, 843, 876, 937, 953
   Severidad: ALTO
   
   Descripción:
   - input() se usa sin try-except para manejar EOFError
   - Causará falla si el programa se ejecuta en modo no interactivo
   - No hay validación de entrada del usuario
   
   Ejemplo del problema:
   ```
   if input("\n📧 ¿Deseas enviar el reporte por correo? (s/n): ").lower() == 's':
   ```
   
   Solución:
   ```
   try:
       respuesta = input("\n📧 ¿Deseas enviar el reporte por correo? (s/n): ").lower()
       if respuesta == 's':
           # continuar
   except EOFError:
       print("No input disponible. Abortando.")
       return
   ```
